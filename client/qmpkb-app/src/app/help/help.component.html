Help

Quantitative Mouse Proteomics Knowledgebase (MouseQuaPro) is Web-based application that hosts experimental data on protein abundances in various mouse tissues measured by mass spectrometry-based targeted MRM proteomics.

MouseQuaPro is hosted at the Proteomics Centre of the University of Victoria and is publicly and freely available via the {{ baseUrl }}.

Searching

MouseQuaPro supports basic and advanced search queries, as well as filtering and retrieving search results via the Web interface or using the dedicated API.

Default search

In the default search option that is available at the front page of MouseQuaPro, users can combine multiple search criteria and terms together to make their search specific (see Figure 1). In addition, it is also possible to search to multiple terms in the same category, like a list of protein accession numbers by separating these with new line (for example by copy-paste from an Excel column), or by a vertical bar (for example A6X935|O08742). By selecting one or more fields, users can restrict the search space and make it faster to retrieve the results. Users are advised to always use the default search option to make the search more focused and faster.



Figure 1. Default search in MouseQuaPro

MouseQuaPro will check and report the presence of peptide assays in the user's protein sequences and report them in three categories. “Present and unique” within the user's protein sequences, “Present but not unique”, and “Not present”. Ideally, the user limits the search space by using additional search fields with required terms when using this option (like limiting the search to a strain, tissue, and specific pathway).

Basic search

Users can perform simple searches using the basic search box that is available on right top in the navigation bar on all pages of MouseQuaPro (see Figure 2). Users can use search term, like– partial – protein name (e.g. Afamin), UniProtKB accession (e.g. O89020), gene name (e.g. Afm), Subcellular localization (e.g. Secreted), peptide sequence (e.g. AAPITQYLK), Molecular pathway(s) name (e.g. Complement and coagulation cascades), Involvement in disease name (e.g. Adiponectin deficiency(ADPND)), GO ID (e.g. GO:0008015), GO term (e.g. blood circulation), GO aspects (e.g. Biological Process), Drug associations ID (e.g. DB05202) , Strain (e.g. NODSCID), Mutant (e.g. Wild), or Panel (e.g. Plasma-1) or Sex (e.g. Male) or Biological matrix (e.g. Mouse Plasma) in the search field, and then pressing 'submit'.



Figure 2. Navigation bar with basic search field.

Result page

Once the search results are on screen, the user can add filters to each column. The results are organized in a simple data table with protein entries in rows. Each row belongs to one protein and reports the peptide assays used/available, the tissues in which the protein concentration was determined, the human ortholog, whether the assay is suitable for the human ortholog, subcellular localization, involvement in biological pathways, associated gene ontology terms, involvement in disease, and association with drugs. An example of the result page with the search term “apolipoprotein” is illustrated in the Figure 3 and can be found using this link: {{ baseUrl }}/results?searchterm=apolipoprotein

Summary statistics

At the bottom of the result page is summary statistics that report on how the results related to the user search term(s) are related to tissues, pathways, gene ontologies, sub cellular localization, and dieses. MouseQuaPro usually report the top 10 in each of these categories except of the tissue in which it report summary statistics numbers on all tissues in the knowledgebase. An example of the summary results obtained with the search term “apolipoprotein” is included in Figure 4 below.

The plot in the summary statistics are interactive and are associated with filtering the column of the results data table. For example if the user wishes to see from the results only those assays measured in a specific tissue like “heart”, or associated with a selected disease like “Bladder Neoplasm”, one can select these directly in the summary statistics plots by clicking at the respective bar in the bar plot. The users can alternatively also insert the filter term they are interested in in the column filter boxes at the top of the data table.



Figure 4. Summary statistics of the search results.

Detailed view page

The detailed view page is protein-specific and includes multiple sections. It has sections on the assay details, measured concentration in tissues, gene expression (from NCBI), and protein features and annotations. Further sections are related to subcellular localization, biological pathways, associated gene ontology terms, known involvement in disease, and known association with drugs. All information in the different section can be downloaded using the corresponding download button. The detailed view page features various interactive plots in the different sections that allow users to investigate the various aspects of the protein and measured concentrations in tissues. For example, in the concentration section users can view boxplots showing all concentrations and determined dynamic ranges according to tissue, strain, and sex. In the protein features section, the user can view the available protein annotation including variations, post-translational modifications, and other features like domains and chains. We adapted ProtVista (Watkins, et al., 2017) to include the surrogate peptide used in our assay. Because human proteins are better annotated than their mouse orthologs, we chose to display the annotations for the mouse protein alongside its human ortholog. Data submission

MouseQuaPro accepts data submission by sharing the raw data, on which we will run our analysis pipeline and then it will be automatically included. As we are not a raw data hosting service like PRIDE, ProteomExhange, or Panorama Web, our intention is to handle submission to MouseQuaPro after depositing the raw data in one of these open proteomics data repositories, which we will download, process with our pipeline, and include it in MouseQuaPro. Please use the data submission form to inform us about the data and we will get in contact with you as soon as we can download the raw data.

MouseQuaPro Implementation

MouseQuaPro is written mainly in Python 2.7. The backend API and Web interface were developed in Django 1.11 and Angular 7.0 framework respectively. The search functionalities were implemented using Elasticsearch 6.2 as the search backend engine.MouseQuaPro integrates protein concentrations with external data from UniProt(Boutet, et al., 2007), Kyoto Encyclopedia of Genes and Genomes KEGG (Kanehisa and Goto, 2000), National Center for Biotechnology Information NCBI(Brown, et al., 2015), QuickGO(Binns, et al., 2009), and DisGeNET(Pinero, et al., 2017). The integration happens in real time by consuming the corresponding Web services of the various resources. Google charts and PlotlyJS were used for visualization. The REST API was built using Django REST framework. Swagger UI was used to generate interactive API documentations.
